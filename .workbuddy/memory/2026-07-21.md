## 用户服务（seeyou-user）搭建

### 配置修正
- `application.yml`：`id-type: auto` → `input`（雪花算法手动赋值）；`logic-delete-field: deleted` → `isDeleted`（匹配 `is_deleted` 字段）
- `pom.xml`：新增 `spring-security-crypto`（BCrypt 加密）、`knife4j-openapi3-jakarta-spring-boot-starter`（Swagger 注解）
- `config/JwtConfig.java`：注入 `JwtUtils` Bean（与网关共用同一 secret）+ `BCryptPasswordEncoder`

### 实体 & Mapper
- `entity/UserInfo.java`：对应 `user_info` 表，`@TableId(type=INPUT)` 雪花 ID，`@TableLogic` 逻辑删除
- `entity/UserPoints.java`：对应 `user_points` 表
- `mapper/UserInfoMapper.java`、`mapper/UserPointsMapper.java`

### DTO / VO
- `dto/RegisterDTO`：username + password + nickname（校验注解）
- `dto/LoginDTO`：username + password
- `vo/LoginVO`：token + userId + username + nickname + avatarUrl
- `vo/UserInfoVO`：用户信息（不含密码）

### Service
- `UserInfoService` / `UserInfoServiceImpl`
  - `register`：检查用户名 → 雪花 ID → BCrypt 加密 → 插入 user_info → 初始化 user_points
  - `login`：查用户 → 校验密码 → 签发 JWT → 返回 LoginVO
  - `getCurrentUserInfo`：从 `UserContext.getUserId()` 取 ID → 查库 → 转 VO

### Controller
- `UserController`：`POST /api/user/register`、`POST /api/user/login`、`GET /api/user/info`
- register/login 在网关白名单内；info 需鉴权

### 整条链路
注册 → 登录拿 token（含 userId/username/role）→ 带 `Authorization: Bearer <token>` 请求 `/api/user/info`
→ 网关 AuthGlobalFilter 解析 JWT → 透传 `X-User-Id`/`X-Username`/`X-User-Role` Header
→ UserContextInterceptor 写入 ThreadLocal → `UserContext.getUserId()`/`getRole()` 取用

### 编译验证
- `mvn -pl seeyou-user -am compile -DskipTests` BUILD SUCCESS

## JWT 加入 role 字段

网关原来只透传 userId/username，补上 role：

- `JwtUtils.createToken(Long, String, Integer role)` — payload 加 `role`
- `JwtUtils.getRole(String token)` — 解析 role
- `CommonConstants.HEADER_USER_ROLE = "X-User-Role"`
- `AuthGlobalFilter` 解析 role → 透传 `X-User-Role` header
- `UserContext.LoginUser` 加 `Integer role` 字段；`UserContext.getRole()`
- `UserContextInterceptor` 从 `X-User-Role` 读取写入 ThreadLocal
- `UserInfoServiceImpl.login` 签发时传入 `user.getRole()`
